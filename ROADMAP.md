# Roadmap

План развития библиотеки `composite_logger` в строгом соответствии с приоритетами.

## v0.0.5: Производительность
- [x] **Асинхронность**: Сделать процесс логирования асинхронным (использование буферизированных каналов и воркеров).

## v0.0.6: Форматирование и Сеть
- [ ] **JSON Formatter**: Реализовать поддержку JSON-формата для консольного и файлового адаптеров.
- [ ] **Telegram Timeouts**: Добавить настраиваемые таймауты для сетевых запросов к Telegram API.

## v0.0.7: Качество и Примеры
- [ ] **Code Coverage**: Настроить автоматический расчет и отображение процента покрытия кода тестами.
- [ ] **База примеров**:
    - [ ] Отдельные примеры для каждого логгера.
    - [ ] Детальный пример использования Composite Logger.
    - [ ] Пример реализации и подключения кастомного адаптера пользователем.

## v0.0.8: Обслуживание
- [ ] **Ротация логов**: Добавить функционал автоматической ротации лог-файлов (Log Rotation).

## v1.0.0: Профессиональная документация
- [ ] **Godoc**: Добавить подробные комментарии ко всем экспортируемым функциям и типам для идеального отображения на `pkg.go.dev`.
